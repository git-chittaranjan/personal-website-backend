Understanding app.UseStaticFiles() and wwwroot in ASP.NET Core

Introduction

ASP.NET Core uses the wwwroot folder to store static files such as HTML, CSS, JavaScript, and images. These files are served to the browser only when the middleware app.UseStaticFiles(); is enabled in the application pipeline.

What Is wwwroot?

The wwwroot folder is the web root of an ASP.NET Core application. Any file placed inside this folder becomes directly accessible via a URL.

MyApp/ │ ├─ Program.cs ├─ Startup.cs (if using older versions) │ ├─ wwwroot/ │ ├─ css/ │ │ └─ site.css │ ├─ js/ │ │ └─ script.js │ └─ images/ │ └─ logo.png
Files inside wwwroot can be accessed like this:
/css/site.css

What Does app.UseStaticFiles(); Do?

ASP.NET Core does not automatically serve static files. You must manually enable the static file middleware in the request pipeline using:

app.UseStaticFiles();

When enabled, ASP.NET Core checks incoming URLs and maps them to files inside wwwroot. If the matching file exists, it is returned directly to the browser without any controller or Razor Page involvement.

Program.cs Example

Below is how static files are usually enabled in a minimal hosting model:

var builder = WebApplication.CreateBuilder(args); var app = builder.Build(); app.UseStaticFiles(); // Enables serving files from wwwroot app.MapGet("/", () => "Hello World!"); app.Run();

How the Browser Accesses Static Files

Example: if you place sample.html inside wwwroot, you can access it directly:

https://localhost:5001/sample.html
